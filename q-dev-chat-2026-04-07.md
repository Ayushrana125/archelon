**`git checkout -b dev`**

- `checkout` — switch to a branch
- `-b` — create a new branch (b = branch)
- `dev` — name of the new branch

So this creates a new branch called `dev` and switches you to it immediately. It's a copy of whatever branch you were on (`main`) at that exact moment.

---

**`git push -u origin dev`**

- `push` — upload your local branch to GitHub
- `-u` — set upstream, meaning link your local `dev` to the remote `dev` permanently. After this, just `git push` is enough — no need to type `origin dev` every time
- `origin` — the name for your GitHub remote (default name)
- `dev` — the branch to push

So this creates the `dev` branch on GitHub and links it to your local `dev`.

---

**In simple terms:**

`git checkout -b dev` → creates the branch on your computer

`git push -u origin dev` → creates the same branch on GitHub

After this, both your computer and GitHub have a `dev` branch that is an exact copy of `main` at this moment.
