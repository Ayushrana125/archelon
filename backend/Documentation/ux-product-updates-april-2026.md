## Overview

This document covers all UX and product changes made after the core platform was stable. These are improvements to the chat experience, embed widget UX, deployment view, and upgrade flow.

---

## 1. Upgrade Modal Redesign

### Before

- Title: "You've used all your tokens"
- Body: "Upgrade your plan to keep chatting with Archelon and unlock more tokens."
- Button: "Upgrade Plan" (dummy, no action)
- Dismiss: "Maybe later"

### After

- Title: **"Keep chatting with Archelon"** — forward-looking, not negative
- Body: "Your free tokens are up. Upgrade to keep chatting and keep your agents active for visitors."
- Button: **"Contact Archelon Team"** — opens Gmail compose directly in new tab with pre-filled enterprise email
- Dismiss: "Dismiss"

### Pre-filled Email

Opens `https://mail.google.com/mail/?view=cm&to=aranixlabs@gmail.com` with:
- Subject: `Archelon Enterprise Plan Inquiry`
- Body: Professional enterprise inquiry — mentions scaling usage, asks for pricing, token limits, SLA, custom deployment options. No mention of free plan or tokens running out.

User just adds their name and hits send.

### Why Gmail URL instead of mailto

`mailto:` opens the system default mail app — could be Outlook, Apple Mail, etc. `mail.google.com/mail/?view=cm` opens Gmail directly in the browser, which is what most users expect.

## 2. Deployments View Redesign

### Before

`DeploymentsView.jsx` was a fake local-state UI:
- Toggle to "enable" agents — didn't persist to DB
- Dummy masked API key (`arch_live_••••••••••••••••••••••••••••••••`)
- Embed script with `arch_live_YOUR_KEY_HERE` — broken if copied
- Widget name field — local state only, not saved

### After

Real deployment overview using `embedStatuses` from `App.jsx` (prefetched on login):
- Shows only **deployed agents** (`enabled === true`)
- Each card shows real live/inactive status from DB
- If agent has a widget name set, shows it below the agent name
- Clicking any agent opens `EmbedModal` directly with prefetched status — no extra API call
- Empty state: "No deployed agents yet. Use the Embed button on an agent to deploy it."

### Props Added

`DeploymentsView` now receives:
- `embedStatuses` — map of `agentId → status` from App.jsx
- `onEmbedStatusChange` — callback to update the map when status changes
- `user` — passed through to EmbedModal

## 3. Embed Widget — Bubble-based Thinking Steps

### Before

Steps were rendered as a static overlay above the input box (`#arch-steps-overlay` div). All steps were created at once and advanced by toggling CSS classes. Not interactive, felt like a loading bar.

### After

Steps appear as real bot chat bubbles in the message area — same avatar, same bubble style as regular messages. They flow naturally from bottom up as the chat progresses.

**Behaviour:**
- Step 1 ("Analyzing your request") appears immediately on send
- For RAG: steps 2-4 appear one by one at **1100ms consistent intervals**
- Each step has the bot avatar + bubble with green pulsing dot + animated `...` dots
- Completed steps fade to gray (done state)
- All step bubbles removed when `clearSteps()` is called on `done` event

**Why consistent intervals matter:** Previously steps advanced as fast as possible — all 4 steps appeared nearly simultaneously. Now each step waits 1100ms before the next appears, giving the user a sense of real work happening.

### Send Button

- Normal state: up arrow (↑)
- Loading state: ring spinner animation (`loading` CSS class on `#archelon-send`)
- Restores to up arrow on `done`/error

## 4. Embed Widget — Pre-chat Screen

Before the first message is sent, the widget shows a pre-chat screen instead of an empty message area:

- Top right: theme toggle (moon/sun) + close button
- Center: agent logo in circle (no border, no background)
- Agent name + green "Online" dot
- Rotating greeting from 5 random phrases
- 3 sample question chips (fetched silently from backend)

Sample questions are generated by a silent RAG call on widget load:
```
"Generate 3 short sample questions users mostly ask from these documents. 
Max 8 words each. Give only in "","","" format, nothing else."
```

Parsed from the `"q1","q2","q3"` format and rendered as clickable chips.

## 5. Embed Widget — FAB Pill Design

### Before

Circular FAB button with teal/blue gradient glow.

### After

Horizontal pill shape:
- Black background (`#0a0a0a`)
- White border (`rgba(255,255,255,0.18)`)
- Deep black box shadow glow
- Left: agent logo clipped to circle (no ring, no background — just the image)
- Right: "Ask {Name}" text (closed) / "Close" text (open)

**Logo handling:** Logo is clipped to circle shape with `overflow: hidden` and `object-fit: cover`. If the logo is a PNG with transparency, only the actual image shows — no white circle edge.

**FAB hidden until info fetch:** FAB starts with `opacity: 0` and `pointer-events: none`. Only becomes visible after `/api/public/info` resolves. This eliminates the flash where the wrong logo appears briefly before the correct one loads.

## 7. Embed Widget — Message Actions

Every bot response has action buttons below the bubble:
- 👍 Thumb up — locks on click, prevents double-vote, turns green when active
- 👎 Thumb down — locks on click, prevents double-vote, turns red when active
- Copy — copies raw text, shows checkmark for 1.5s, turns green when copied

Thumb up and thumb down are mutually exclusive — clicking one locks both.

---

## 8. Embed Widget — Scroll Behaviour

- Scroll-to-bottom button appears when user scrolls up more than 80px from bottom
- New bot messages scroll to the top of the message (not bottom) — so the start of the answer is visible
- `wrap.scrollIntoView({ behavior: 'smooth', block: 'start' })` on each new bot message

---

## 9. EmbedModal — Click-to-Edit Settings Card

Settings card is read-only by default. Clicking anywhere on the card enters edit mode:
- All fields become editable
- Teal border appears on the card
- "Cancel" button appears in header (click outside also cancels)
- "Save Changes" button appears at bottom (only enabled when `hasChanges` is true)
- `originalSettings` snapshot taken when edit mode starts — `hasChanges` compares against it

This prevents accidental edits and makes it clear when changes are pending.

---

## 10. EmbedModal — "Widget is live" State

When the modal opens for an already-enabled agent (`apiKey === 'masked'`):
- Left panel shows a pulsing green dot with "{Name} is live" text
- Not the iframe preview (that only shows when key was just generated)

This gives clear visual confirmation that the widget is deployed and active.
